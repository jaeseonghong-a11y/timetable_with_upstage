■ 이 문서 세트는 무엇인가
Upstage AI Document Builders Challenge 출품작 개발의 "외부화된 기억(하네스)".
매번 대화로 맥락을 재설명하지 않도록, 진행상황·결정·검증·규칙·과제를 파일로 분리 저장한 것.
재성이 이 파일들을 프로젝트 파일에 넣어두면, 다음 세션의 AI가 맥락을 그대로 이어받는다.

■ 파일 읽는 순서 (새 세션 시작 시)
1) 00_프로젝트_현황_요약   ← 지금 어디까지 왔나 (제일 먼저)
2) 05_미해결_과제           ← 다음에 뭘 해야 하나
3) 필요 시 01~04 참조:
   01_의사결정_로그   = 왜 이렇게 정했나 (결정 근거)
   02_기술검증_기록   = 뭐가 되고 뭐가 안 되나 (삽질 방지)
   03_아이디어_후보_평가 = 후보 목록·평가 (메인 엎어질 때 대안)
   04_규칙과_지켜야할것  = 지켜야 할 원칙·안전선

■ 갱신 방법 (재성이 AI에게 시킬 때)
- "현황 업데이트해줘" → 00 갱신 (큰 진전 반영)
- "방금 결정한 거 로그에 남겨줘" → 01에 [D-번호] append
- "이 검증결과 기록해줘" → 02에 append (실패도 반드시 남김)
- "이 과제 해결됐어" → 05에서 [해결→XX]로 옮기고 해당 파일에 결론 이관
각 파일 상단 "업데이트 규칙"을 따른다. 과거 이력은 지우지 않고 보존(번복은 표시).

■ 갱신 후 재반영 워크플로우
1. AI가 파일 수정 → 2. 재성이 다운로드 → 3. 프로젝트 파일에서 기존 것 교체
→ 이렇게 하면 항상 최신 맥락이 프로젝트에 유지됨.

■ 핵심 원칙 (이 하네스의 철학)
- 실패도 기록한다 (같은 삽질 반복 방지 = 하네스의 존재이유).
- 결정의 "이유"를 남긴다 (나중에 "왜 이렇게 했지?"를 잃지 않도록).
- 번복된 결정도 지우지 않는다 (사고의 궤적 보존).
- 이 파일들은 "매 응답에 항상 적용"이 아니라 "필요할 때 참조"하는 자료.
  (항상 적용할 행동규칙은 프로젝트 지침에, 이 파일들은 프로젝트 파일에)
