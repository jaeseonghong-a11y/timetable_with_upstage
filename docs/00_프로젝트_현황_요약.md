■ 프로젝트 한 줄 정의
Upstage AI Document Builders Challenge 출품작. 데모데이 2026-07-25.
"흩어진 정보를 실제로 쓸 수 있는 서비스로" — Upstage Parse/Extract/Solar 중 1개 이상 필연 사용 필수.

■ 현재 메인 (확정 — 옵션 A 채택, D-12)
"강의계획서를 읽어 학기 부담까지 예측하는 시간표 조합 추천 서비스" (성균관대 학부생 대상)
- 개설강좌 수집(scraper)은 최소 시딩. 서비스 심장은 강의계획서 PDF를 Upstage Document Parse/Extract로
  구조화해 평가방식·과제비중·시험일정·수업유형을 뽑아내는 것.
- GLS 기수강·졸업요건 문서를 Parse해 사용자의 남은 요건과 과목 후보를 연결한다.
- 결정론적 엔진이 필수과목·책가방·분반의 모든 유효 조합을 만들고 Solar는 검증된 조합의 설명·정렬만 담당한다.
- 기존 시간표앱의 충돌 체크를 넘어 개인 졸업 맥락과 강의계획서 근거를 함께 쓰는 추천 서비스다.

       전공 강좌를 즉시 조회. 개발용 JSON 버튼 제거, 과목 검색·이수구분 표시·검토 중 기수강 자동 제외

       요청 시점에 조회하고 전공 후보와 합침. 전공/교양 탭·영역 선택·과목 검색 구현
[완료] 시간표 결과 시각화 — 텍스트 과목 나열을 월~금 08:00~22:00 블록 주간표로 교체하고
       조합별 접기/펼치기 및 모바일 내부 가로 스크롤 구현
[완료] 기본정보 입력 보강 — 학과·전공·트랙을 대학·학부 대분류별로 묶은 검색 선택창,
       1~6학년과 초과학기 선택 구현
[완료] Phase 6b 강의계획서 분석 UI — 사용자 PDF→서버 전용 Document Parse→명시 평가 항목·시험 표기 표시
[완료] 공개 교과과정 수집 기반 — 학과·전공·융합트랙 코드로 선택 범위 전체 교과목 조회 및 라이브 검증
[완료] 강의계획서 PDF 접근성 검증 — 수집된 INTRO_URL은 접근제한 HTML 반환; 사용자 PDF 업로드로 전환 (P6)
[완료] GLS 학사문서 서버 파이프라인 — PDF/PNG/JPG→Document Parse→Solar→AcademicProfile
       런타임 검증까지 구현. 원본·전체 Parse 결과는 반환/저장하지 않고 항상 draft로 시작
[완료] GLS 학사문서 확인 UI — 기수강/졸업요건 행 수정·추가·삭제, 재수강 토글, 검토 항목별 확인,
       confirmed 전환 구현. 졸업요건 캡처 Ctrl+V 붙여넣기 지원, 확정 뒤 수정하면 자동으로 draft 복귀
       졸업요건 카드 개별/전체 접기와 문서 결과 축소 지원, 확정 완료 시 편집 영역 자동 접기
       펼친 요건도 5열 압축 편집으로 표시하고 수강 중 학점 입력은 숨김, 반복 검토 사유는 그룹화
       사용하지 않는 수강중 0값·알려진 DS 중복 안내는 비차단 처리, 확인 항목 0개면 검토 패널 숨김
       기준/취득 미달 문구는 오류에서 제외하고 숫자 요건의 잔여학점으로 충족/미충족 직접 판정
       전체 확인/해제를 지원하고 C/L 중복·결정론적 분류·기준초과·공동규칙 안내는 검토에서 제외
[완료] 졸업요건 행 정규화 — Solar의 숫자문자열·한글 키·숫자형 원문 값은 안전하게 변환해 행을
       보존하고, `6 / 0` 같은 복합값만 행 단위 review로 남김(16개 행 전체 폐기 회귀 수정).
       Solar가 앞 3행만 반환해도 Document Parse 표에서 교양·DS·균형교양 누락 행을 순서대로 보완
       균형교양은 3개 영역 중 최소 2개 영역·그룹 합계 6학점의 공동 규칙으로 구조화
[완료] 수강/취득과목 원본 정규화 — 학수번호 숫자 3~4자리와 한글/분리 학기 표기를 허용하고,
       Document Parse 표의 코드·과목명 쌍을 기준으로 Solar 누락을 보완. 32개 표 행에 합쳐진 두 과목도
       분리해 원본 실측 33과목·학기 33개·검토 사유 0건을 확인. 분석 직후 `earned + exclude`는 과목
       선택에서 숨기고, 사용자가 재수강 예정으로 바꾼 `retake`만 다시 표시

- 대주제: 흩어진 정보를, 쓸 수 있는 서비스로.

  사용자 제약을 검증 가능한 파이프라인으로 결합해 결정론적 유효 시간표를 만든다.
