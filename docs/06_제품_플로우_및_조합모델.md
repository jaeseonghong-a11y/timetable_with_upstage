# 제품 플로우 및 누락 없는 조합 모델

최종 갱신: 2026-07-14

## 1. 권장 화면 흐름

시간표는 데스크톱에서 오른쪽에 처음부터 고정해 두고, 왼쪽 단계 입력에 따라 즉시 갱신한다.
모바일에서는 입력 단계 아래에 접히는 미리보기로 둔다.

1. **기본 정보**: 소속 학과·전공·트랙, 입학연도, 현재 학년, 주 캠퍼스
   - 전체 학번은 받지 않는다. 졸업요건 분기에 필요한 4자리 입학연도만 받는다.
   - 조회 학년도·학기를 함께 선택하면 Next 서버가 해당 학과 범위의 실제 전공 개설강좌를 즉시
     불러온다. 사용자는 scraper JSON 파일이나 성대 학과코드를 알 필요가 없다.
2. **기수강 과목**: GLS 수강/취득과목 PDF·이미지 업로드 또는 수동 입력
   - Upstage Document Parse로 표를 통일하고, 사용자가 결과를 확인·수정한다.
   - 기수강 과목은 기본 제외하되 `재수강` 토글로 다시 후보에 포함한다. 기록 자체를 삭제하지 않는다.
3. **졸업요건**: 졸업요건충족현황 캡처/PDF 업로드
   - Document Parse가 표를 읽고 Solar가 미충족 영역을 구조화한다. AI 결과는 확정값이 아니라 사용자가
     승인해야 하는 추천 입력값이다.
4. **필수 선택**: 이번 학기에 반드시 들을 과목을 고른다. 과목을 고르면 가능한 분반 중 하나가 선택된다.
5. **고정 일정**: 시간표에 수업이 들어가면 안 되는 구간을 `기타 일정`으로 넣고 이름을 자유롭게 정한다.
6. **책가방**: 관심 과목을 하나 이상의 책가방에 넣고, 각 책가방마다 최소·최대 선택 수를 정한다.
7. **성향**: 공강 희망일, 아침 회피, 연강 회피, 캠퍼스 혼합 허용 등을 정한다.
8. **결과**: 결정론적 엔진이 가능한 조합을 생성·검증하고 Solar가 짧은 근거를 붙여 정렬한다.
   결과 카드를 누르면 오른쪽 시간표에 표시한다.

### 구현된 편집 방식 (2026-07-14)

- 과목 목록 위에서 새 과목을 담을 곳을 `필수 과목` 또는 원하는 `선택 그룹`으로 지정한다.
- 기본 위치는 `선택 그룹 1`이며 기본 규칙은 후보 중 정확히 1과목이다. 따라서 여러 과목을 체크해도
  전부 필수가 되지 않고 과목별 대안 조합이 생성된다.
- 선택 그룹은 여러 개 만들고 이름과 최소·최대 과목 수를 수정할 수 있다. 그룹을 삭제하면 그 그룹에
  있던 과목의 선택과 분반 설정도 함께 해제되며, 과목 목록에서 다시 선택할 수 있다.
- 선택된 과목 이름을 펼치면 실제 분반별 교수·시간·캠퍼스·수업형태를 확인하고 허용할 분반만 남긴다.
  허용 분반이 0개이면 임의 분반을 고르지 않고 명시적 오류를 표시한다.
- 펼친 과목의 `이 과목 선택 해제` 버튼으로 전공·교양 과목을 현재 그룹에서 직접 제거할 수 있다.

## 2. 조합 모델

### 데이터 계층

- `Subject`: 학수번호 단위 과목. 같은 과목의 여러 분반을 소유한다.
- `Section`: 실제 개설 분반. 시간·캠퍼스·수업방식이 여기에 속한다.
- `RequiredSubject`: 모든 결과에 반드시 포함할 과목.
- `ChoiceBag`: 과목 집합과 `minSubjects`·`maxSubjects`를 가진 선택 제약.
- `ExcludedSubject`: 기수강 과목 등 기본 제외 대상. 재수강 토글 시 제외 집합에서 제거한다.
- `FixedEvent`: 알바·동아리·이동 등 이름을 바꿀 수 있는 고정 시간 구간.

### 생성 순서

1. 필수과목을 모두 넣는다.
2. 각 책가방에서 최소~최대 개수의 **과목 조합**을 빠짐없이 만든다.
3. 선택된 각 과목에서 정확히 한 개의 **분반**을 고르는 데카르트 곱을 만든다.
4. 고정 일정·수업 간 시간 충돌을 제거한다.
5. 캠퍼스·수업방식·사용자 성향 필터를 적용한다.
6. 남은 조합에만 Solar 설명·정렬을 적용한다.

현재 UI는 1~3단계를 `selection-plan.ts`와 시간표 엔진으로 결정론적으로 수행한다. 전체 시간표가
안전 한도 500개를 넘으면 결과를 자르지 않고 조건을 좁히라는 오류를 낸다.

Solar가 2~3단계를 직접 수행하면 일부 경우가 빠져도 검증할 수 없으므로 금지한다. Solar는 유효 조합을
만드는 주체가 아니라, 이미 검증된 조합을 설명하고 사용자 요구에 맞게 정렬하는 역할이다.

## 3. 구조 오류 점검 결과

### 같은 과목의 분반을 한 그룹에 넣는 경우

올바른 구조다. `자료구조`라는 과목을 한 번 선택한 뒤 01·02·03분반 중 정확히 하나를 고르게 해야 한다.
분반을 서로 다른 책가방 과목으로 넣으면 같은 학수번호를 두 번 수강하는 잘못된 조합이 생긴다.

### 책가방마다 무조건 하나만 고르는 경우

표현하지 못하는 조합이 생긴다. 한 가방에서 두 과목을 듣거나 아무것도 듣지 않는 경우를 만들 수 없기
때문이다. 따라서 각 가방은 `0~1`, `1`, `2`, `1~3`처럼 최소·최대 선택 수를 가져야 한다.

### 같은 과목을 여러 책가방에 넣는 경우

한 과목이 두 조건을 동시에 충족하는지, 두 번 선택해야 하는지 의미가 모호하다. MVP에서는 중복 배치를
오류로 막고 한 곳으로 이동시키게 한다. 일반적인 모든 과목 조합은 하나의 큰 가방에 후보를 넣고 원하는
총 선택 수를 지정하면 빠짐없이 만들 수 있다. 영역별 최소 이수 조건은 서로 겹치지 않는 가방으로 나눈다.

### 모든 경우의 수를 실제로 만들 수 있는가

위 구조가 표현한 범위에서는 가능하다. 조합 엔진이 과목 부분집합과 분반 곱을 결정론적으로 열거하기
때문이다. 다만 후보가 많으면 수가 지수적으로 커지므로 무제한 결과를 메모리에 모두 쌓을 수는 없다.
안전 한도를 넘으면 임의로 일부를 추천처럼 반환하지 않고, 책가방 선택 수·학점 범위·시간 제약을 더
정하도록 명시적으로 중단한다. 이후에는 지연 생성과 제약 기반 가지치기로 확장한다.

### 아직 별도 규칙이 필요한 항목

- 동일과목/대체과목/재수강 인정 관계
- 선수·동시수강 과목
- 최대·최소 신청학점과 과목별 학점
- 공동개설(C/L) 과목의 중복 학수번호 정규화
- 캠퍼스 간 실제 이동시간

## 4. 캠퍼스·온라인 수업 규칙

- 기본 결과는 한 시간표에 인문사회·자연과학 두 물리 캠퍼스가 섞이지 않게 한다.
- 사용자가 `캠퍼스 혼합 보기`를 켜면 섞인 결과도 보여 주되 같은 날 연속 수업은 이동시간을 검증한다.
- `온라인(사전제작)`은 비동기 수업으로 보고 물리 캠퍼스·이동시간 계산에서 제외한다.
- 실시간 온라인 수업은 시간 충돌에는 포함하지만 물리 캠퍼스 이동 계산에서는 제외한다.

## 5. 교과과정 데이터 원칙

- 성균관대 대표 홈페이지의 공개 `학과/교과목 검색`과 교육과정 팝업을 1차 소스로 사용한다.
- 최신 학과·전공·융합트랙 코드는 기존 GAIA 학과목록 조회 결과를 사용한다. 중앙 이름 검색에 노출되지
  않는 융합트랙도 코드를 교육과정 팝업에 전달하면 교과목을 받을 수 있음을 확인했다.
- 사용자가 선택한 학과·전공·트랙만 조회하고 전체 데이터를 상시 미러링하지 않는다.
- 교육과정 목록은 추천 후보군이고, 실제 해당 학기 개설 여부·분반·시간은 개설강좌 API와 교차 확인한다.
- 입학연도별 졸업요건은 공개 교과목 목록만으로 확정하지 않고 사용자가 올린 GLS 문서를 우선한다.
