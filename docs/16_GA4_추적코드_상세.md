# GA4 + Clarity 추적 코드 상세 (재성님 전달용)

13_윤서_피드백_실행계획.md 3절·7절의 지표를 실제로 심는 코드.
React/Next.js(App Router) 기준으로 작성 — 실제 프로젝트 구조에 맞게 조정 필요.
`G-XXXXXXXXXX`(GA4 측정 ID), `CLARITY_ID`는 윤서가 콘솔에서 만들어 전달.

> ⚠️ 개인정보 원칙: 이벤트 파라미터에 **파일명, 문서 내용, 학번 등 식별 정보 절대 금지.**
> 아래 코드의 파라미터(이벤트명, step 번호, 문서 종류, 에러 유형, 소요시간)만 전송한다.

const CLARITY_ID = "xxxxxxxxxx";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ko">
      <body>
        {children}

        {/* GA4 */}
        <Script
          src={`https://www.googletagmanager.com/gtag/js?id=${GA_ID}`}
          strategy="afterInteractive"
        />
        <Script id="ga4-init" strategy="afterInteractive">
          {`
            window.dataLayer = window.dataLayer || [];
            function gtag(){dataLayer.push(arguments);}
            gtag('js', new Date());
            gtag('config', '${GA_ID}');

        </Script>

        <Script id="clarity-init" strategy="afterInteractive">
          {`
            (function(c,l,a,r,i,t,y){

            })(window, document, "clarity", "script", "${CLARITY_ID}");
          `}
        </Script>
      </body>

  );

## 1. 공용 헬퍼 — 모든 이벤트가 거쳐가는 함수

```ts
// lib/analytics.ts
type EventParams = Record<string, string | number | undefined>;

export function track(eventName: string, params: EventParams = {}) {
  if (typeof window === "undefined") return;
  const gtag = (window as any).gtag;
  if (typeof gtag !== "function") return; // 광고차단기 등으로 미로드 시 무시
  gtag("event", eventName, params);
}
```

## 2. 단계별 이탈률 — `step_enter`

단계 전환이 state로 관리된다는 가정 (한 페이지 앱).

```tsx
// 단계를 관리하는 최상위 컴포넌트
import { useEffect } from "react";
import { track } from "@/lib/analytics";

const STEP_NAMES: Record<number, string> = {
  1: "basic_info",       // 기본정보 입력
  2: "doc_upload",       // 학사문서 업로드·분석
  3: "course_select",    // 과목 선택·조합 생성
  4: "ai_recommend",     // AI 시간표 추천
  5: "save_image",       // 이미지 저장
};

function TimetableFlow() {
  const [step, setStep] = useState(1);

  useEffect(() => {
    track("step_enter", { step, step_name: STEP_NAMES[step] });
  }, [step]);

  // ...
}
```

**GA4에서 보는 법:** 탐색 → 유입경로 탐색 분석 → 단계에 `step_enter`를
`step=1`~`5` 조건으로 5개 배열 → 구간별 이탈률이 바로 나옴.

## 3. 입력 항목별 이탈률 — `field_focus` / `field_complete` / `form_abandon`

```ts
// lib/analytics.ts 에 추가
let lastFieldFocused = "";
let lastStep = 0;
let completed = false; // timetable_created(완주) 후 true → 이탈로 안 침

export function onFieldFocus(fieldName: string, step: number) {
  lastFieldFocused = fieldName;
  lastStep = step;
  track("field_focus", { field_name: fieldName, step });
}

export function onFieldComplete(fieldName: string, step: number, value: string) {
  if (!value.trim()) return; // 빈 값이면 완료 아님
  track("field_complete", { field_name: fieldName, step });
}

export function markCompleted() {
  completed = true;
}

// 앱 마운트 시 딱 1회 호출
export function initAbandonTracking(getCurrentStep: () => number) {
  document.addEventListener("visibilitychange", () => {
    if (document.visibilityState === "hidden" && !completed) {
      // gtag는 자동으로 sendBeacon을 써서 페이지 이탈 중에도 전송됨
      track("form_abandon", {
        last_field_focused: lastFieldFocused || "(none)",
        step: getCurrentStep() || lastStep,
      });
    }
  });
}
```

```tsx
// 각 입력 필드에 연결 (예: 기본정보 단계)
<input
  name="admission_year"
  onFocus={() => onFieldFocus("admission_year", 1)}
  onBlur={(e) => onFieldComplete("admission_year", 1, e.target.value)}
/>
<select
  name="major"
  onFocus={() => onFieldFocus("major", 1)}
  onChange={(e) => onFieldComplete("major", 1, e.target.value)}
>
```

주의: `visibilitychange`는 **탭 전환에도 발생**하므로 한 세션에서 여러 번
찍힐 수 있음. 분석할 때는 "세션의 마지막 `form_abandon`"만 보면 된다.
(모바일에서는 `beforeunload`가 안 잡히는 경우가 많아 이 방식이 표준)

**GA4에서 보는 법:** `form_abandon` 이벤트를 `last_field_focused` 측정기준으로
쪼개면 이탈 시점의 필드 분포가 나옴. `field_complete`를 필드 순서대로 퍼널에
넣으면 완료율 낙차가 보임.

## 4. 문서 분석 성공률·실패 유형·소요시간 — `doc_upload_start` / `doc_parse_success` / `doc_parse_fail`

```ts
// 업로드 핸들러 (기존 업로드 로직에 계측만 추가)
import { track } from "@/lib/analytics";

type DocType = "enrollment" | "graduation"; // 수강내역 / 졸업요건

async function handleDocumentUpload(file: File, docType: DocType) {
  track("doc_upload_start", { doc_type: docType });
  const t0 = performance.now();

  try {
    const result = await parseDocument(file); // 기존 Upstage 호출
    track("doc_parse_success", {
      doc_type: docType,
      duration_ms: Math.round(performance.now() - t0),
    });
    return result;
  } catch (err) {
    track("doc_parse_fail", {
      doc_type: docType,
      error_type: classifyParseError(err),
      duration_ms: Math.round(performance.now() - t0),
    });
    throw err;
  }
}

// 에러를 "유형"으로만 분류 — 에러 메시지 원문(파일명 포함 가능)은 보내지 않음
function classifyParseError(err: unknown): string {
  const msg = err instanceof Error ? err.message : String(err);
  if (/format|type|지원/i.test(msg)) return "invalid_format";
  if (/size|large|용량/i.test(msg)) return "file_too_large";
  if (/timeout|시간/i.test(msg)) return "timeout";
  if (/network|fetch/i.test(msg)) return "network";
  if (/4\d\d|5\d\d|api/i.test(msg)) return "api_error";
  return "unknown";
}
```

## 5. AI 추천 클릭·대기시간 — `ai_recommend_click` / `ai_recommend_done`

```ts
async function handleRecommend(weights: Weights) {
  track("ai_recommend_click");
  const t0 = performance.now();

  const result = await requestRecommendation(weights); // 기존 Solar 호출

  track("ai_recommend_done", {
    duration_ms: Math.round(performance.now() - t0),
  });
  return result;
}
```

**GA4에서 보는 법:** `duration_ms`를 맞춤 측정항목(아래 8절)으로 등록하면
평균/분포를 볼 수 있음. "p50이 5초를 넘는가"가 판단 기준.
세션당 `ai_recommend_click` 횟수가 3회 이상이면 첫 추천 불만족 신호.

## 6. 가중치 조정 사용률 — `weight_adjust`

슬라이더는 드래그 중 수십 번 발생하므로 디바운스 필수.

```tsx
import { useMemo } from "react";
import { track } from "@/lib/analytics";

function useTrackWeightAdjust() {
  return useMemo(() => {
    const timers: Record<string, ReturnType<typeof setTimeout>> = {};
    return (weightType: string) => {
      clearTimeout(timers[weightType]);
      timers[weightType] = setTimeout(
        () => track("weight_adjust", { weight_type: weightType }),
        800
      );
    };
  }, []);
}

// 사용
const trackWeight = useTrackWeightAdjust();
<Slider
  onChange={(v) => {
    setMorningWeight(v);
    trackWeight("no_morning_class"); // 가중치 종류별 이름
  }}
/>
```

## 7. 완주·저장 — `timetable_created` / `timetable_save`

완주 기준은 **저장 버튼 클릭이 아니라 "완성된 시간표가 화면에 표시됨"**이다.
(시간표를 다 만들고 저장만 안 누른 사람을 이탈로 잘못 집계하지 않기 위함)
3초 유지 조건을 붙여 결과 화면을 스쳐 지나간 경우는 제외한다.

```tsx
// 완성된 시간표를 렌더링하는 컴포넌트에서
import { useEffect, useRef } from "react";
import { track, markCompleted } from "@/lib/analytics";

function CompletedTimetable({ timetable }: { timetable: Timetable | null }) {
  const fired = useRef(false); // 세션당 1회만 발사

  useEffect(() => {
    if (!timetable || fired.current) return;
    const timer = setTimeout(() => {
      fired.current = true;
      track("timetable_created");
      markCompleted(); // 완주 인정 → 이후 떠나도 form_abandon으로 안 잡힘
    }, 3000); // 3초 이상 화면에 떠 있어야 완주로 인정
    return () => clearTimeout(timer);
  }, [timetable]);

  // ...
}
```

저장 클릭은 별도의 **만족·실사용 의향 지표**로 계속 추적한다:

```ts
function handleSaveImage() {
  track("timetable_save"); // 저장률 = timetable_save ÷ timetable_created
  // ... 기존 이미지 저장 로직
}
```

## 8. GA4 콘솔 설정 (코드 아님 — 윤서가 GA4 관리 화면에서 등록)

커스텀 파라미터는 등록해야 보고서에서 측정기준으로 쓸 수 있다.
**관리 → 맞춤 정의 → 맞춤 측정기준 만들기** (범위: 이벤트):

| 측정기준 이름 | 이벤트 파라미터 |
|---|---|
| step | `step` |
| step_name | `step_name` |
| field_name | `field_name` |
| last_field_focused | `last_field_focused` |
| doc_type | `doc_type` |
| error_type | `error_type` |
| weight_type | `weight_type` |

**맞춤 측정항목 만들기** (범위: 이벤트, 단위: 밀리초):

| 측정항목 이름 | 이벤트 파라미터 |
|---|---|
| duration_ms | `duration_ms` |

배포 링크에는 UTM을 붙여서 채널별 비교. 코드 변경은 필요 없음 — gtag.js가 첫 페이지뷰에서
URL의 `utm_source`/`utm_medium`/`utm_campaign`을 자동으로 읽어 세션 소스/매체로 기록한다.
채널마다 아래 링크 중 해당하는 것을 그대로 복사해서 배포하면 된다(15_윤서_피드백_실행계획.md
2장의 배포 채널 기준):

| 채널 | 배포 링크 |

| 구글 설문지(주력, 설문 안내문 안의 링크) | `https://timetable-with-upstage.vercel.app/?utm_source=googleform&utm_medium=survey` |
| 학과 단톡방 | `https://timetable-with-upstage.vercel.app/?utm_source=kakao&utm_medium=classchat` |
| 동아리방 | `https://timetable-with-upstage.vercel.app/?utm_source=kakao&utm_medium=clubchat` |
| 오픈채팅방 | `https://timetable-with-upstage.vercel.app/?utm_source=kakao&utm_medium=openchat` |
| 에브리타임 게시판(추가로 쓸 경우) | `https://timetable-with-upstage.vercel.app/?utm_source=everytime&utm_medium=post` |

GA4에서 채널별로 보는 법: 보고서 → 획득 → 트래픽 획득에서 "세션 소스/매체"를 `utm_source`/
`utm_medium` 조합으로 나눠서 보여준다. 별도 등록 없이 기본 제공된다.

## 9. 검증 방법 (배포 전 필수)

1. 개발 환경에서 `gtag('config', GA_ID, { debug_mode: true })`로 바꾸거나
   크롬 확장 "Google Analytics Debugger"를 켠다.
2. GA4 → 관리 → **DebugView**를 열어둔 채 사이트를 처음부터 끝까지 한 바퀴 돈다.
3. 체크리스트: `step_enter` 1~5가 순서대로 찍히는가 / 필드 포커스·블러 시
   `field_focus`·`field_complete`가 찍히는가 / 탭을 벗어나면 `form_abandon`에
   `last_field_focused`가 담겨 있는가 / 문서 업로드 성공·실패 각각 확인 /
   추천 실행 시 `duration_ms`가 그럴듯한 값인가 / 완성된 시간표를 3초 이상
   보고 나면 `timetable_created`가 찍히는가(그 전에 떠나면 안 찍혀야 함) /
   `timetable_created` 이후에는 탭을 벗어나도 `form_abandon`이 **안** 찍히는가 /
   저장 버튼 클릭 시 `timetable_save`가 별도로 찍히는가.
4. Clarity 대시보드에서 본인 세션 녹화가 보이는지 확인.
