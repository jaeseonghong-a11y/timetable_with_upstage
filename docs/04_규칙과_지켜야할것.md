1. 대회 필수 규칙 (어기면 탈락/감점)
═══════════════════════════════════════
[필수] Upstage Parse/Classify/Extract/Solar 중 1개 이상을 "필연적으로" 사용.
       → 곁가지가 아니라 서비스 심장에 있어야 함. "왜 Upstage가 필연인가"에 한 줄로 답 가능해야.
[필수] "챗봇에 물어보는 것과 무엇이 다른가"에 답할 수 있어야 함.
[필수] 단순 요약/조회를 넘어 Action(비교·추천·자동작성)으로 이어질 것.
[금지] Claude API로 문서처리 대체 = 대회 부적합 (조원案 드롭 사유). 문서처리는 Upstage로.
[금지] 크롤링/웹검색이 서비스의 심장이 되는 구조 = 조원案 드롭 함정. 크롤링은 조연까지만.

2. 데이터 확보 안전선 (법적 리스크 관리)
═══════════════════════════════════════
근거: 잡코리아 판례 — 문제는 "크롤링 자체"가 아니라 "DB를 통째 복제해 경쟁서비스로 재게시".
      판례 원칙: 접근권한 판단은 기술적보호조치(로그인)·이용약관 등 객관적 사정 기준.

[안전] 로그인 없이 열리는 공개페이지 조회 (성대 개설강좌 = 로그인 불필요, 안전측)
[안전] robots.txt 준수 (성대 kingoinfo = robots.txt 404, 명시적 차단 없음)
[안전] 사용자가 조회한 범위만 그때그때 처리 (세션 단위)
[준수] 요청 간격 두기 (서버 부담 회피)
[금지] 로그인 뒤(GLS 등 인증 뒤) 데이터 자동 크롤링 = 정보통신망 침입 리스크
[금지] 전 과목 DB 통째 미러링 후 상시 게재 = 잡코리아형 리스크
[금지] 강의평(에브리타임 등) 크롤링 = 약관위반 + 개인 저작물
[좁은 예외·2026-07-22 D-23] 사용자가 이 서비스에서 명시적으로 선택한 최대 12개 과목에 한해서,
       사용자가 설치한 브라우저 확장프로그램이 에타 **검색 결과 화면의 `lecture/view/{id}` 이동 URL
       한 개**를 과목명·교수명으로 매칭해 그 사용자 기기에만 캐시할 수 있다. 이는 강의평 화면·본문·
       별점·댓글을 읽지 않는 탐색 보조이며, 비공개 API/로그인 쿠키 접근·서버 전송·전 과목 사전 수집·
       공유 DB화는 모두 계속 금지한다. 동명이거나 불확실하면 자동 선택하지 않고 사용자가 에타 화면에서
       직접 고른다.
[주의] 개인정보·전화번호 노출 금지
[유의] 위는 판례기반 일반론이지 법률자문 아님. 배포·상업화 시 전문가 확인 별도.

[방식] 한번에 큰 코드 X. 동작확인 가능한 작은 단위로 쪼개기. 단계마다 "무엇을 확인할지" 명시.
[방식] 에러 시: 원인 → 수정코드 → 재확인 방법 순.
[파일] HTML/CSS/JS 역할분리. 스타일값은 CSS :root 변수로. 폴더명 영문소문자.
