- 로그인: 불필요 (로그아웃 상태에서 200 OK 확인)

- ★ 강의계획서 URL (lcms.skku.edu/... 형태로 응답에 포함) ← Upstage Parse로 넘길 링크가 공짜로 딸려옴

■ ★ 핵심 발견: 메뉴마다 조회조건이 전부 다르다
좌측 메뉴 6종을 F12로 전수 확인한 결과, 조회 파라미터 구조가 제각각:
| 메뉴 | 조회조건 |
| 학사-전공과목 | 년도학기·캠퍼스구분·주관학부(대학)·학과전공 |
| 학사-교양/기타/DS | 년도학기·캠퍼스구분·영역(고전명저·글로벌·소통과사고·창의·미래SW/AI·인간문화·사회역사·자연과학기술·일반선택·기타·DS기반·DS심화) |
| 일반대학원과목 | 년도학기·주관학부·학과전공 (캠퍼스구분 없음) |
| 도전학기과목 | 년도학기·개설캠퍼스·검색어 |
| 전체과목 검색 | 년도학기·개설캠퍼스·조회구분 |

※ 교양은 영역버튼 2단계 방식(숫자=영역별 과목수)이라 전공보다 수집 한 단계 복잡.

2. 수집 방법 3가지 — 실패/성공 기록

■ 방법 A: 순수 API 직접호출 (Python requests, 단국대式)
- 상태: ✓✓ 검증 완료 (2026-07-11 갱신). ★ 이게 최선. C안 관찰로 완전한 레시피 확보함.
- 1차 실패(초기): 내가 SSV 요청 body를 손으로 대충 만들어 fetch → ErrorCode:-1.
  원인: 넥사크로 SSV 규약(헤더·구분자·필수 파라미터)을 부정확하게 재현.
- 해결: C안(엔진 통신 후킹)으로 엔진이 실제로 보내는 요청 전문을 관찰 → A안의 정답 레시피 확보.
- ★ 확보된 완전 명세 (전공과목 기준):
  · 엔드포인트: POST https://kingoinfo.skku.edu/gaia/E_NHSSU900020M/selectMain.do
    (교양은 다름: E_NHSSU900010M/selectMain03.do — 메뉴마다 엔드포인트 다름 재확인)
  · 요청 헤더: Content-Type: text/xml / Accept: application/xml, text/xml, */*
              X-Requested-With: XMLHttpRequest / X-NX-Content-Type: 2 / cache-control: no-cache
  · 요청 본문(SSV, 구분자 RS=0x1e): 앞부분에 쿠키/GA값들 나열 후 핵심 파라미터:
      YEAR=2026 / TERM=15(여름,10=1학기 추정) / HAKGWA_CD=316901(★학과코드, 이것만 바꾸면 다른 학과)
      / CAMPUS_GB=1(인문사회,2=자연과학) / ROAD_MAP=% / HAK_JIBJUNG=0
      / _FIRST_OUT_DS_NM=dsGrdMain / _TRANSACTION_ID=selectMain
  · ★ 넥사크로가 표준 XMLHttpRequest로 나감 (초기에 "XHR 안 씀"이라 본 건 오판 — 필터를
    selectMain03으로만 걸어 못 잡았던 것). 그래서 Python requests로 그대로 재현 가능.
- ★ 응답 SSV 스키마 (self-describing, 파싱 쉬움):
    SSV:UTF-8 → ErrorCode:int=0(성공) → ErrorMsg:string=SUCCESS → Dataset:dsGrdMain
    → _RowType_ 다음 컬럼정의(US=0x1f 구분) → 이후 데이터 행들.
    주요 컬럼: GAESUL_YEAR/GAESUL_TERM / HAKSU_NO_BUNBAN(학수번호-분반) /
    GWAMOK_NAME·GWAMOK_ENG_NAME(과목명 국/영) / HAKJUM(학점) / PER_NAME(교수) /
    GYOSI_NAME(시간·교시) / HYUNGTAE(강의실/형태) / ISU_NAME(이수구분) / SUUP_TYPE_NM(수업형태) /
    CAMPUS_NM(캠퍼스) / ★INTRO_URL(강의계획서 URL — Upstage Parse용)
- 개발 시 남은 확인: HAKGWA_CD 학과코드 목록(드롭다운 API로) / 쿠키 실제 필수 여부(Python서 테스트) /
  TERM 코드 매핑(10·15·20=1학기·여름·2학기 등).

■ ★ 추가 검증 완료 (2026-07-11, fetch 직접호출로 실증)
- 쿠키 필요 여부: ✅ 불필요 확정. credentials:"omit"(쿠키 완전제거)로 호출해도 ErrorCode:0 SUCCESS.
  → Python requests에서 세션/쿠키 없이 그대로 재현 가능. A안 서버독립실행 최종 확증.
- TERM 코드 매핑 (경영학과316901·2026 기준 실측):
  · TERM=10 → 103개 (1학기, 정규)
  · TERM=15 → 20개  (여름학기, 계절)
  · TERM=20 → 107개 (2학기, 정규)
  · TERM=25 → 0개   (겨울학기, 미개설)
- 교양(selectMain03.do) 직접호출 검증: ✅ 쿠키 없이 ErrorCode:0 성공.
  · 엔드포인트: POST /gaia/E_NHSSU900010M/selectMain03.do
  · 핵심 파라미터: YEAR/TERM/HAKSU_NO(영역코드 예 GEDG001)/CAMPUS_GB(2=자연)/ROAD_MAP=%/HAK_JIBJUNG=0
    /_FIRST_OUT_DS_NM=dsGrdMain03/_TRANSACTION_ID=selectMain03
  · 실측: GEDG001=41과목, GEDG002=14과목 (영역코드마다 과목수 다름)
  · 응답 데이터셋명: dsGrdMain03
- ★ 전공 vs 교양 응답 스키마 비교: 핵심 필드 공통 → 파서 공유 가능.
  공통 컬럼: GAESUL_YEAR/GAESUL_TERM/CAMPUS_NM/GWAMOK_NAME/GWAMOK_ENG_NAME/HAKJUM/
  PER_NAME/GYOSI_NAME/HYUNGTAE/SUUP_TYPE_NM/ISU_NAME/★INTRO_URL(강의계획서)
  (교양 전용: SEMINAR_SUBJECT/LANG_CD 등 일부 추가 / 전공 전용: HAKWI_NAME/ABEEK 등)
- 실데이터 확인 예(교양 GEDG001): "영어쓰기/English Writing, 매튜아담, 화09:00-10:15【27423】,목10:30-11:45"
- 미완: HAKGWA_CD(전공 학과코드) 전체 목록 — [해결됨] 2026-07-11.
  · 방법: 학과콤보(cboHakgwa) DOM id로 넥사크로 경로 역추적 → 콤보 채우는 API 특정.
  · 학과목록 API: POST /gaia/E_NHSSU900020M/selectBizType04.do
    파라미터: JOJIK_GB=31 / TONG_HAKBU=(대학코드4자리) / YEAR / TERM / _FIRST_OUT_DS_NM=dsHakgwa
    / _TRANSACTION_ID=selectBizType04. 쿠키 불필요(credentials:omit 성공).
  · 대학코드(TONG_HAKBU) 22개로 반복호출 → 전 학과 132개 수집 완료.
  · 결과는 별도 파일 "성대_학과코드_전체.txt"에 저장 (학과코드/학과명/소속대학/대학코드).
  · 건축학과=316307, 경영학과=316901 등. 학과코드는 대체로 대학코드(4자리)+2자리 구조.
  · 참고: 런타임에도 cboHakgwa.getInnerDataset()로 학과목록 직접 읽기 가능(하드코딩 대안).

■ 방법 B: 브라우저 자동화 + 렌더링 추출
- 상태: ✓ 성공 (검증 완료). Claude-in-Chrome MCP로 실제 실증. 단 A안 확보로 우선순위는 밀림.

- 실제 추출 성공 예: 자연과학캠퍼스 전공과목 목록(과목명·학수번호·교수·학점·수업형태) DOM에서 좌표기반 셀 추출.
- 단점: 매번 브라우저 띄우고 렌더링 대기 → 느리고 무거움. 전과목 수집엔 비효율.

  좌표클릭 안 맞아도 확실히 조회 트리거됨.

■ 방법 C: 넥사크로 엔진 통신 후킹 (레시피 확보 수단) — ✓ 검증 완료
- 상태: ✓ 성공 (2026-07-11). A안 레시피를 뽑는 도구로 역할 완수.
- 성공한 방법: nexacro._startCommunication 후킹 → 엔진이 보내는 TransactionItem/URL 관찰.
  이어서 저수준 XMLHttpRequest.prototype.open/send 후킹(필터를 '.do' 전체로) → 요청/응답 SSV 전문 캡처.

5. 넥사크로(Nexacro) 기술 메모 (재도전용)
═══════════════════════════════════════
- 통신: transaction() 함수로 서버 송수신. 포맷은 SSV/XML/BIN/ZLIB 지원.
- SSV 응답 예: "SSV:UTF-8[RS]ErrorCode:int=0[RS]ErrorMsg:string=SUCC[RS]..." (구분자 RS=0x1e, US=0x1f)
- 응답 성공 시 ErrorCode:int=0, 실패 시 -1.
- 파싱 시 특수구분자(0x1e 레코드, 0x1f 컬럼)로 컬럼셋/로우 분리.
- 우리 브라우저 자동화 시 유용했던 접근: DOM에서 좌표(getBoundingClientRect) 기반 leaf div 텍스트 추출.
